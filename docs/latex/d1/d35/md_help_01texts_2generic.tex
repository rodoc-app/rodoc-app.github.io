 This is a brief introduction to this application for organizing family trees. Each {\bfseries{family tree}} is saved and organized in a {\bfseries{project}}.\hypertarget{md_help_01texts_2generic_autotoc_md7}{}\doxysection{\texorpdfstring{Project}{Project}}\label{md_help_01texts_2generic_autotoc_md7}
Each {\bfseries{project}} is stored in a specific directory on a filesystem, assuming that the entire subtree of the project root directory should not be used for different purposes.

Various kinds of entities can be used throughout the project to express all the intended information related not just to persons and their relations.\hypertarget{md_help_01texts_2generic_autotoc_md8}{}\doxysection{\texorpdfstring{Persons}{Persons}}\label{md_help_01texts_2generic_autotoc_md8}
Each family tree primarily contains a list of {\bfseries{persons}}, each having basic information about themselves, such as a {\bfseries{name}}, {\bfseries{surname}}, etc. Each person can also have multiple user-\/defined {\bfseries{tags}}.\hypertarget{md_help_01texts_2generic_autotoc_md9}{}\doxysection{\texorpdfstring{Relations}{Relations}}\label{md_help_01texts_2generic_autotoc_md9}
To bind two particular {\bfseries{persons}} together, we use the concept of a {\bfseries{relation}} to record their real-\/world interconnection.\hypertarget{md_help_01texts_2generic_autotoc_md10}{}\doxysection{\texorpdfstring{Events}{Events}}\label{md_help_01texts_2generic_autotoc_md10}
{\bfseries{Persons}} may also attend various types of events, such as weddings, allowing these to be associated with a {\bfseries{place}}, {\bfseries{date}}, as well as allowing the persons to participate in specific roles.\hypertarget{md_help_01texts_2generic_autotoc_md11}{}\doxysection{\texorpdfstring{Files}{Files}}\label{md_help_01texts_2generic_autotoc_md11}
Finally, each {\bfseries{person}} can have an unlimited number of physical files of arbitrary types, logically divided into three groups of {\bfseries{files}}, {\bfseries{media}}, and {\bfseries{notes}}.
